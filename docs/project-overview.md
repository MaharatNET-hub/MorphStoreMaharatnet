# توصيف المشروع — نظام تصميم متعدّد الأشكال (MorphStore)

> ملخّص المستند الفني الأصلي المرفق. المصدر الكامل للتفاصيل والجداول هو المستند نفسه؛
> هذا الملف يلخّص القرارات المعتمدة في التنفيذ الفعلي داخل هذا المستودع.

## 1. الهدف

بناء نظام تصميم متكامل (Design System / Template Library) على ووردبريس، وليس متجراً
واحداً ثابتاً. الأدمن يجهّز عدّة أشكال جاهزة لكل عنصر (هيدر، بانر، كارت منتج، شيك أوت...)،
ومدير المتجر (Shop Manager) يختار الشكل الأنسب لكل عميل من صفحة إعدادات واحدة، دون أي
صلاحية تعديل على التصميم نفسه.

## 2. البنية التقنية (Stack)

| المكوّن | الدور |
|---|---|
| WordPress | نظام إدارة المحتوى |
| WooCommerce | محرّك المتجر |
| Astra (Free) | الثيم الأساسي |
| Elementor (Free) | بناء الصفحات والأقسام |
| Advanced Custom Fields (Free) | حقول اختيار الأشكال ولوحة الإعدادات |

## 3. آلية اختيار الأشكال (المعتمدة في هذا المستودع)

الطريقة (ب) من المستند الأصلي — لوحة إعدادات ACF مركزية:

- صفحة إعدادات واحدة (`أشكال المتجر`) تحت `manage_woocommerce` (صلاحية Shop Manager
  الافتراضية).
- حقل Select واحد لكل عنصر (12 عنصر بالمجمل)، كل الحقول مُسجَّلة بالكود
  (`theme/astra-child-maharatnet/inc/acf-fields.php`) فلا حاجة لأي إعداد يدوي.
- خريطة ربط (`inc/template-map.php`) تحوّل كل اختيار إلى ID قالب Elementor حقيقي.
- دالة موحّدة `ms_render_store_part()` تطبع القالب المطابق أينما استُدعيت
  (هيدر/فوتر/كارت المنتج تلقائياً، وبقية العناصر عبر Shortcode `[store_part field="..."]`).

## 4. عناصر المتجر والأشكال المتاحة

| العنصر | حقل ACF | عدد الأشكال |
|---|---|---|
| الهيدر | `header_style` | 4 |
| البانر / الهيرو | `hero_style` | 5 |
| التصنيفات | `categories_style` | 4 |
| أقسام المنتجات | `product_sections_style` | 2 |
| كارت المنتج | `card_style` | 4 |
| صفحة المتجر / الأرشيف | `shop_style` | 2 |
| صفحة المنتج المفرد | `single_product_style` | 4 |
| صفحة البراندات | `brands_style` | 2 |
| السلة | `cart_style` | 3 |
| الشيك أوت | `checkout_style` | 2 |
| آراء العملاء | `testimonials_style` | 3 |
| الفوتر | `footer_style` | 3 |

القيم النصية الكاملة لكل اختيار موجودة في `theme/astra-child-maharatnet/inc/acf-fields.php`.

## 5. طريقة الربط حسب نوع العنصر

- **هيدر / فوتر / كارت المنتج**: ربط مباشر بالكود (ملفات `header.php`, `footer.php`,
  `inc/hooks-woocommerce.php`) — يعمل تلقائياً فور رفع القوالب وربط الـ ID.
- **هيرو / تصنيفات / أقسام منتجات / آراء عملاء / براندات**: عناصر تُدرَج داخل صفحات
  Elementor عبر Shortcode `[store_part field="hero_style"]` في أي مكان بالصفحة.
- **صفحة المتجر / المنتج المفرد / السلة / الشيك أوت**: قوالب كاملة الصفحة، الأفضل بناؤها
  عبر JetWooBuilder وربطها شرطياً بإضافة Dynamic Conditions مقارنةً بقيمة حقل ACF
  (الطريقة أ في المستند الأصلي). الكود هنا يضيف كلاس `body` يعكس الاختيار الحالي
  (مثال: `body.shop-style-shop-02`) لتسهيل هذا الربط الشرطي أو أي تخصيص CSS إضافي.

## 6. تقييد صلاحيات Shop Manager

`inc/role-permissions.php` يحجب عن دور `shop_manager` صلاحيات تعديل قوالب Elementor
وإعدادات الثيم والتخصيص، ويُبقيه قادراً فقط على الوصول لصفحة «أشكال المتجر» عبر
`manage_woocommerce` التي يملكها افتراضياً.

## 7. قائمة الإضافات

راجع `bin/plugins-wp-org.txt` (تُركَّب تلقائياً عبر `setup-server.sh`) و
`bin/plugins-manual.txt` (تحتاج تحميل يدوي — إضافات Crocoblock وغيرها).

## 8. خطوات التنفيذ المتبقية بعد رفع الكود

هذا المستودع يجهّز **البنية البرمجية الكاملة** (الثيم، الحقول، منطق التبديل، الصلاحيات،
سكربتات النشر). الخطوات التالية تبقى يدوية داخل ووردبريس نفسه بعد التنصيب، لأنها تصميم
مرئي وليست كوداً:

1. تصميم كل شكل (Variant) في Elementor وحفظه كـ Template.
2. نسخ ID كل قالب ولصقه في `theme/astra-child-maharatnet/inc/template-map.php`.
3. (اختياري) تصميم قوالب Archive/Single عبر JetWooBuilder وربطها بإضافة Dynamic
   Conditions لعناصر المتجر/المنتج المفرد/السلة/الشيك أوت.
4. مراجعة صفحة «أشكال المتجر» في لوحة التحكم واختيار الأشكال الافتراضية لكل عميل.
